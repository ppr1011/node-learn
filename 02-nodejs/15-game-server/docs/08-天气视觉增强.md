## 0. 背景与设计原则

天气由**服务端权威**生成:`src/spawn/definitions/weather.ts` 用运行时随机源定时重掷一份
全局状态 `{ kind, intensity }` 并广播,保证多端一致(见 [核心难点](01-核心难点.md) 的生成框架)。
客户端**只拿到这两个字段**,其余所有视觉细节自己"编"出来。

升级前,客户端天气只是三段最朴素的绘制:雨=斜线段、雪=白圆点、雾=一层半透明矩形。
本次增强遵循两条与项目一脉相承的原则:

1. **程序化优先(范式 A)**:延续本 Demo「零图片资源、代码生成美术」的主线
   (见 [客户端视觉系统](06-客户端视觉系统.md))。带宽/内存成本为 0,且效果天然随 `intensity` 缩放。
2. **贴图可选(范式 B)**:额外留一个贴图接入点作为对照,演示"资源管线"这条路,
   **默认不启用**,缺资源自动回退程序化——两种范式并存、可对比。

> 关键取舍:为什么不直接引入 AI 生图 / 序列帧贴图作为主方案?因为那会破坏本项目
> 「一个种子生成万物、零资源依赖」的教学叙事。贴图更适合作为"如果要学资源管线"的补充分支。

## 1. 分层景深:让粒子"有远近"

朴素做法里所有雨滴/雪花一个样,画面是"平"的。增强版给每个粒子一个**景深** `d ∈ [0,1]`,
用它同时驱动多个视觉量,近处粒子长/快/亮/粗,远处短/慢/淡/细——一个变量撑起纵深感:

```javascript
function newRainDrop(W, H) {
  const d = 0.35 + Math.random() * 0.65;   // 景深
  return { x: ..., y: ..., d,
           len: 7 + d * 15,   // 近处更长
           spd: 8 + d * 13 }; // 近处更快
}
// 绘制时:近处更亮更粗
ctx.strokeStyle = `rgba(175,200,232,${(0.18 + 0.42 * p.d) * fx.ramp})`;
ctx.lineWidth   = 0.6 + p.d * 1.1;
```

雪花同理:`d` 驱动半径 `r`、下落速度 `spd` 与透明度,形成多层视差(parallax)。

## 2. 对象池 + 强度驱动数量

沿用 [粒子系统](06-客户端视觉系统.md#3-粒子系统) 的对象池思想,但把"目标数量"交给
`强度 × 淡入进度` 决定,每帧只做增量增删,不重建数组:

```javascript
const fit = (pool, target, make) => {
  while (pool.length < target) pool.push(make(W, H));
  if (pool.length > target) pool.length = target;   // 截断即回收
};
if (kind === 'rain') fit(fx.rain, Math.floor(scale * 340), newRainDrop);
```

- 小雨(`intensity` 低)→ 稀疏几十滴;暴雨 → 三百多滴。**同一份代码,强度自适应。**
- `scale = intensity * fx.ramp`,`ramp` 是天气切换后的淡入进度,见第 6 节。

## 3. 地面涟漪:top-down 的正确溅射

俯视视角下,雨点是"打在地面各处",而不是"落到屏幕底边"。所以溅射涟漪**散布全屏随机位置**,
用压扁的椭圆环模拟俯视透视,扩散并淡出:

```javascript
if (p.y > H) {                                  // 雨滴落地
  if (Math.random() < 0.5) spawnRipple(Math.random() * W, Math.random() * H);
  p.y = -p.len; p.x = ...;                       // 回收到顶部
}
// 涟漪:椭圆(rx, ry=rx*0.4)描边,ry 压扁 → 俯视透视
ctx.ellipse(s.x, s.y, s.r, s.r * 0.4, 0, 0, Math.PI * 2);
```

涟漪数量设了上限(60),避免暴雨时无限堆积拖垮帧率。

## 4. 闪电:随机事件 + 全屏闪光 + 折线

只在**较强降雨**(`intensity ≥ 0.55`)时启用。用一个帧倒计时 `strikeCd` 随机触发落雷:

```javascript
function updateLightning(it) {
  if (it < 0.55) { fx.flash = 0; fx.bolt = null; return; }
  if (--fx.strikeCd <= 0) {
    fx.flash = 1;                                  // 全屏闪光,随后衰减
    fx.strikeCd = 150 + Math.floor(Math.random() * 450); // 下次:~2.5~10s
    fx.bolt = Math.random() < 0.6 ? makeBolt() : null;   // 60% 概率带可见折线
    fx.boltLife = 6;
  }
  fx.flash *= 0.86;                                // 指数衰减,模拟一闪而过
}
```

- **闪光**:`rgba(220,228,255, flash*0.5)` 铺满全屏,`*= 0.86` 让它快速回落。
- **折线**:从屏幕顶部向下随机游走生成折点,用 `shadowBlur` 加辉光。

## 5. 体积雾:流动的柔光团

朴素雾是一层不动的半透明灰,很"死"。增强版改为**若干缓慢横向漂移的径向渐变柔光团**,
飘出右边界从左边回卷,叠加出流动、不均匀的体积感:

```javascript
for (const b of fx.fog) {
  b.x += b.spd;
  if (b.x - b.r > W) b.x = -b.r;                  // 回卷
  const g = ctx.createRadialGradient(b.x, b.y, 0, b.x, b.y, b.r);
  g.addColorStop(0, `rgba(214,218,228,${b.a * it * fx.ramp})`);
  g.addColorStop(1, 'rgba(214,218,228,0)');        // 边缘完全透明 → 柔和无硬边
  ctx.fillStyle = g; ctx.arc(b.x, b.y, b.r, ...);
}
```

## 6. 全局色调分级 + 平滑过渡

真实天气会改变**整个环境的色温**。`ambientTint()` 在最底层铺一层全屏色,给画面定调:

| 天气 | 色调 | 意图 |
|------|------|------|
| rain | 偏蓝、压暗 `rgba(18,26,48, …)` | 阴冷、压抑 |
| snow | 清冷偏亮 `rgba(150,172,210, …)` | 干净、寒意 |
| fog  | 灰(由 `drawFog` 负责) | 朦胧 |
| clear| 恒定暖阳 `rgba(255,236,180,0.05)` | 温暖日照 |

**平滑过渡**:天气切换时若色调/粒子瞬间满值会"啪"地跳变。用一个淡入进度 `fx.ramp`(0→1,
约 0.8s)统一乘到色调 alpha 与粒子目标数量上,让新天气"渐渐登场":

```javascript
if (kind !== fx.kind) { /* 清空旧池 */ fx.ramp = 0; fx.kind = kind; }
fx.ramp = Math.min(1, fx.ramp + 0.02);
```

## 7. 范式 B:可选贴图接入(优雅降级)

留一个贴图接入点演示资源管线,**默认关闭**,且缺资源时自动回退程序化——demo 始终可运行:

```javascript
const WEATHER_SPRITES = { /* snow: 'assets/weather/snowflake.png' */ };

function weatherSprite(key) {
  if (!(key in WEATHER_SPRITES)) return null;
  const img = new Image();
  img.onerror = () => { _spriteCache[key] = null; };  // 加载失败 → 回退
  img.src = WEATHER_SPRITES[key];
  _spriteCache[key] = img; return img;
}
const spriteReady = (img) => !!img && img.complete && img.naturalWidth > 0;

// 雪的渲染分支:就绪就画贴图(带旋转),否则画程序化圆点
if (spriteReady(spr)) { ctx.rotate(p.spin); ctx.drawImage(spr, ...); }
else                  { ctx.arc(p.x, p.y, p.r, ...); ctx.fill(); }
```

素材来源、许可、序列帧动画进阶,见 [`client/assets/weather/README.md`](../client/assets/weather/README.md)。
推荐优先选 **CC0**(公共领域免署名)资源,如 [Kenney Particle Pack](https://kenney.nl/assets)。

## 8. 渲染时机

天气层在 [render 循环](06-客户端视觉系统.md#5-平滑摄像机) 里,**屏幕震动 `restore()` 之后**绘制——
所以它是纯屏幕空间的,不随摄像机移动/震动,符合"天气是覆盖在世界之上的氛围"这一直觉:

```javascript
ctx.restore();      // 结束屏幕震动(世界坐标系)
drawWeather(W, H);  // 屏幕空间:色调 → 粒子 → 闪电,叠在最上层
```

## 小结

| 增强点 | 手段 | 关键收益 |
|--------|------|----------|
| 景深分层 | 每粒子 `d` 驱动 长/速/亮/粗 | 平面 → 纵深 |
| 强度自适应 | 目标数量 = `intensity × ramp` | 一份代码,大小雨雪通吃 |
| 地面涟漪 | 全屏随机 + 压扁椭圆 | 符合 top-down 透视 |
| 闪电 | 随机倒计时 + 全屏闪 + 折线 | 暴雨戏剧张力 |
| 体积雾 | 漂移柔光团回卷 | 死雾 → 流动雾 |
| 色调分级 | 底层全屏 tint + `ramp` 淡入 | 环境色温 + 无跳变 |
| 贴图可选 | `WEATHER_SPRITES` + 优雅降级 | 程序化/资源双范式对照 |

全部改动集中在 `client/index.html` 的 Weather 章节,**服务端协议零改动**——
再次印证「服务端管权威状态,客户端管表现」的分层边界。
