击杀敌人 → 掉落随机武器 → 走过自动拾取装备 → 不同武器改变攻击并播放专属动画。整条链路完全沿用本项目**服务端权威**的物件管线（config → 定义 → GameWorld → 协议 → 客户端渲染），与敌人系统同构。

## 1. 权威边界

和树、石、敌人一样：**能影响战斗结果的都在服务端**。武器数值、掉落判定、拾取判定、装备状态全在服务端；客户端只负责渲染掉落物、播放攻击动画，并存一份**只读**的武器数值副本用于 HUD 显示与动画时长。客户端改不了自己的伤害。

## 3. 掉落管线

```
玩家攻击命中敌人(CombatSystem)
  └─ 敌人 HP≤0 → 广播 ENEMY_DEAD + 安排复活
       └─ GameWorld.spawnWeaponDrop(enemy)
            ├─ Math.random() < WEAPON_DROP_CHANCE(0.7)?      掉落几率
            ├─ rollWeaponDrop(luck)                           按权重加权随机
            │    · common 权重恒定,rare×luck,epic×luck²
            │    · luck 由敌人种类决定:demon 1.7 / skeleton 1.1 / slime 0.8(强敌出好货)
            └─ new WeaponDrop(kind, x, y, TTL, GRACE) → drops 表 → 广播 ITEM_SPAWN
```

`WeaponDrop`（`src/core/WeaponDrop.ts`）继承 `Entity` 拿全局唯一 id，带 `pickableAt`(拾取宽限)与 `expiresAt`：45s 无人拾取则自然消失，避免地图堆满掉落。

## 4. 拾取与装备

拾取判定放在 `GameWorld.tick → updateDrops(now)`，每 tick：

1. **TTL 清理**：`now ≥ expiresAt` → 删除 + 广播 `ITEM_PICKUP { dropId, byPlayerId: null }`（null 表示自然消失）。
2. **拾取**：先跳过仍在**拾取宽限期**（`now < pickableAt`,落地 700ms 内)的掉落——保证它先在地上可见;宽限期过后,遍历存活玩家,命中 `WEAPON_PICKUP_RADIUS`(26px) 内即被其 `equip` → 删除 + 广播 `ITEM_PICKUP { dropId, byPlayerId, weapon }`。

掉落物数量少且有 TTL 上限，因此 spawn/pickup 走**全局广播**（与天气一致），不引入 AOI 分发的复杂度。死亡不清空武器（复活保留，demo 更友好）。

## 5. 攻击方向与动画协议

为了让「挥砍弧 / 突刺 / 飞行物」有方向，服务端给玩家加了 `facing`（弧度）：移动时按移动向量更新（`MovementSystem`），攻击时朝目标（`CombatSystem`）。

新增 `s_attack` 消息在**每次通过冷却的攻击**时广播（命中与否都发，空挥也有动作）：

```ts
{ attackerId, weapon, dir, range, hit }
```

命中反馈仍走原有的 `DAMAGE` / `ENEMY_HIT`。这样动画（挥击表现）与结算（掉血数字/粒子）解耦，且由服务端回声统一驱动 → **多端看到的挥击方向一致**。

## 6. 客户端渲染

`client/index.html` 新增 `Weapons & Drops` 章节：

- **掉落物**：地面阴影 + 按稀有度上色的脉动光环 + 武器贴图上下浮动（`sin` bob），epic 额外撒火花；小地图上也标出稀有度色点。
- **贴图**：`assets/weapons/*.png`（Kenney Tiny Dungeon，CC0），`imageSmoothingEnabled = false` 放大保持像素风；缺文件时回退 `getProcWeapon` 程序化图标（延续「范式 A/B」）。
- **主角贴图**：玩家不再是圆圈,改用 `assets/player/hero.png`(Kenney 骑士 tile_0087),`drawPlayer` 按 `facing` 水平翻转、受击闪白,身份区分改由脚下身份色圈承担(自身绿/他人按 id);缺文件回退圆圈。
- **攻击动画**（`drawWeaponAttack`，按 `weapon` + `dir` + 进度 `t` 分支）：徒手**有方向的挥拳**(沿 `dir` 出拳→收回 + 命中火花)、匕首双段短刺、铁剑横挥月牙、巨剑旋风、斧/锤过顶下劈（命中触发 `shockwaves` 震波环 + 屏震 + 尘土）、法杖释放 `projectiles` 飞行法球（到达迸发魔法环）。
- **HUD**：顶栏显示当前武器名（稀有度配色）；自己脚下画攻击冷却环。

## 7. 素材来源（CC0）

武器/敌人/主角贴图均取自 [Kenney Tiny Dungeon](https://kenney.nl/assets/tiny-dungeon)（**CC0 1.0**）。该包不含弓/矛，因此武器阵容按包内实际含有的近战 + 法杖设计，保证美术风格统一。tile 映射见 `client/assets/weapons/README.md`、`client/assets/player/README.md`。

## 8. 扩展思考

- **武器耐久 / 掉落归属**：可给 `WeaponDrop` 加 `ownerId` + 归属保护时间，避免被抢。
- **稀有度词缀**：在 `WeaponStats` 上叠加随机词缀（吸血、暴击），掉落时 roll。
- **投射物命中判定上服务端**：当前 `bow/staff` 的飞行物是纯客户端表现（伤害仍是瞬时命中）。若要真实弹道，需把投射物做成服务端实体并做碰撞——参考敌人的实体化路径。
